# 硬件需求指南（NeuralVSR）

> 两阶段：训练阶段（4090 就够） + 评估阶段（需要 80GB）

---

## 阶段 1：训练（Week 1 Day 3-4）

### 推荐配置

| 项目 | 规格 | 原因 |
|------|------|------|
| **GPU** | **RTX 4090 24GB** | GNN 只有 50K 参数，batch=32 绰绰有余 |
| **镜像** | PyTorch 2.6 / CUDA 12.8 / Python 3.10 | 同之前 |
| **数据盘** | ≥30GB | 训练数据 1M pairs ~ 10GB |
| **时长** | 10 小时 | 1M 样本训练 + 调参 |

### AutoDL 选择

搜 "RTX 4090"，选按时计费（~¥2.5/h），**不用选 A100**。

---

## 阶段 2：合成数据生成（Week 1 Day 3）

### 可以用无卡模式省钱

| 项目 | 规格 |
|------|------|
| **GPU** | **无卡模式（CPU-only）** |
| **CPU** | 任意 |
| **费用** | ¥0.1/h |
| **时长** | 5-10 小时 |

合成数据生成不需要 GPU。用 AutoDL 的「无卡模式」开机只收 ¥0.1/h。

**操作**：开实例时不选 GPU，或者现有实例「关机」后选「无卡开机」。

---

## 阶段 3：ISPD2005 评估（Week 1 Day 5-7 + Week 2 消融）

### 小电路（adaptec1/3, bigblue1）— RTX 4090 够用

24GB 能跑的电路：
- adaptec1 (543 macros)
- adaptec3 (723 macros)
- bigblue1 (560 macros)

**用 Week 1 Day 3-4 训练的同一个 4090 实例**就行。

### 大电路（adaptec2/4, bigblue2/3/4）— 必须 A100 80GB

#### 为什么必须升级 GPU

| 电路 | Macros | 边数 | 24GB 情况 |
|------|--------|------|----------|
| adaptec2 | 566 | 18,938 | **OOM** |
| adaptec4 | 1,329 | 28,864 | **OOM** |
| bigblue2 | 23,084 | 144,446 | **OOM** |
| bigblue3 | 1,298 | 32,900 | **OOM** |
| bigblue4 | 8,170 | 223,314 | **OOM** |

显存瓶颈是 **guided sampling 时的 gradient descent** + **GAT edge attention**，和边数强相关。

#### AutoDL 选择

| 选项 | 费用/小时 | 是否推荐 |
|------|----------|---------|
| **A800 80GB** | ~¥22-25 | ✅ **首选**（便宜 20%，对单卡来说和 A100 完全一样） |
| **A100 80GB** | ~¥28-30 | ✅ 可选（如果 A800 没货） |
| **H100 80GB** | ~¥50+ | ⚠️ 太贵没必要（训练只要 0.5h） |
| **A100 40GB × 2** | ~¥25 | ❌ 需要改多卡代码 |
| **RTX 4090 24GB** | ~¥2.5 | ❌ **已验证 OOM**，5/8 电路跑不了 |

**A800 vs A100 对比（单卡场景）**：

| 指标 | A100 80GB | A800 80GB | 对我们的影响 |
|------|-----------|-----------|------------|
| 显存 | 80GB HBM2e | 80GB HBM2e | 一样（唯一重要指标） |
| 算力 | 19.5 / 624 TFLOPS | 19.5 / 624 TFLOPS | 一样 |
| 内存带宽 | 2039 GB/s | 2039 GB/s | 一样 |
| NVLink | 600 GB/s | 400 GB/s | 无影响（我们单卡） |

**推荐：A800 80GB 单卡**。5 小时足够（生成数据 1.5h + 训练 0.5h + 评估 2h + 缓冲 1h）。

## AutoDL 开实例操作（两种场景）

### 场景 1：训练阶段（Week 1 Day 1-6）

1. 登录 AutoDL
2. 新建实例，选：
   - GPU：RTX 4090 24GB × 1
   - 镜像：PyTorch 2.6 / CUDA 12.8 / Python 3.10
   - 数据盘：50GB
3. 复制登录指令 + 密码
4. 按 `docs/autodl_ssh_setup.md` 配置
5. `git pull` 同步最新代码

### 场景 2：大电路评估（Week 1 Day 7）

**选项 A：升级现有实例**（推荐）
1. AutoDL 控制台 → 现有 4090 实例 → 「升级配置」
2. 选 A100 80GB
3. 实例 Host/Port 不变，数据盘保留
4. 直接跑实验

**选项 B：新建 A100 实例**
1. 新建实例选 A100 80GB
2. 数据盘要挂载之前的（选「使用已有数据盘」）
3. 按 SSH 教程重新配置连接

## 省钱技巧

| 技巧 | 节省 |
|------|------|
| 合成数据用无卡模式 | ~¥20 |
| 训练期用按量计费，跑完立刻关机 | ~¥30 |
| A100 只用在必需的大电路评估 | ~¥200（对比全程用 A100） |
| 数据盘保留（~¥0.5/天），实例释放 | 保证不重新下 checkpoint / 数据 |
| 消融实验用 4090（不需要 A100） | ~¥150 |

---

## 你需要现在就做的事

1. **保持现有的 4090 实例**（或重开一个）— 用于训练 + 小电路评估
2. **Week 1 Day 7 准备好 A100** — 提前看 AutoDL 的 A100 是否充足（有时候晚上更好租）
3. **数据盘保留**（关机时选「保留数据盘」），避免每次重下 checkpoint/ISPD2005

## 时间 vs 预算的 trade-off

| 策略 | 时间 | 预算 |
|------|------|------|
| 保守（全用 4090，只跑 3 电路） | 2 周 | ¥100 |
| 标准（4090 训练 + A100 评估） | 2 周 | ¥400 |
| 激进（H100 并行，多配置同时跑） | 1.5 周 | ¥800 |

论文目标是标准策略，留 1 天 buffer。
